# Swish — قبعات ستريت وير

متجر إلكتروني كامل لبيع القبعات موجَّه للسوق الجزائري:
- **صفحة هبوط بتصميم Liquid Glass** (أسود/رمادي فاخر) باللغة العربية (RTL)، بخط حضري (Bebas Neue) وأيقونات SVG.
- **كاروسيل** لعرض الموديلات (سحب/Swipe) مع زر **«اختر»** يُعبّئ الفورم مباشرةً بالموديل المختار.
- صفحة قصيرة جدًا: **المنتج + فورم الطلب فقط** — سريعة التحميل.
- **الدفع عند الاستلام** (Cash on Delivery) — بدون أي دفع مسبق.
- **الأسعار بالدينار الجزائري (دج / DZD)**.
- **توصيل لكل 58 ولاية** (إلى المنزل أو مكتب التوصيل / Stop Desk).
- **لوحة تحكم (Back Office)** لإدخال المنتجات وتتبّع الطلبات وتغيير حالتها.
- **تكامل Meta Pixel** (Facebook/Instagram Ads) مع الأحداث القياسية.

---

## 🚀 التشغيل

> المتطلبات: [Node.js](https://nodejs.org) الإصدار 18 أو أحدث.

```bash
npm install
npm start
```

ثم افتح المتصفح على:

| الصفحة | الرابط |
|--------|--------|
| المتجر (صفحة الهبوط) | http://localhost:3000 |
| لوحة التحكم | http://localhost:3000/admin |

لتغيير المنفذ: `PORT=8080 npm start` (أو على ويندوز PowerShell: `$env:PORT=8080; npm start`).

## 📈 إعداد Meta Pixel

1. ادخل إلى لوحة التحكم → **الإعدادات**.
2. في صندوق **Meta Pixel**، ألصق معرّف البيكسل (Pixel ID) من
   [مدير أحداث فيسبوك](https://business.facebook.com/events_manager) واضغط **حفظ البيكسل**.
3. سيُحقن البيكسل تلقائيًا في صفحة الهبوط، وتُرسَل الأحداث القياسية:
   - `PageView` — عند فتح الصفحة
   - `ViewContent` — عند عرض المنتج
   - `InitiateCheckout` — عند بدء تعبئة الطلب
   - `Purchase` و `Lead` — عند تأكيد الطلب (بقيمة وعملة `DZD`)

## 🧱 البنية

```
.
├── server.js              # خادم Express (المسارات + الـ API)
├── src/
│   ├── db.js              # طبقة تخزين JSON (كتابة ذرية)
│   ├── auth.js            # مصادقة المسؤول (scrypt + كوكي HMAC)
│   ├── wilayas.js         # الـ58 ولاية + أسعار التوصيل الافتراضية
│   └── seed.js            # بيانات أولية (منتجات + إعدادات)
├── public/                # الموقع العام (صفحة الهبوط)
│   ├── index.html
│   ├── css/style.css
│   ├── js/app.js
│   └── uploads/           # صور المنتجات المرفوعة
├── admin/                 # لوحة التحكم
│   ├── login.html · dashboard.html · products.html · orders.html · settings.html
│   ├── css/admin.css
│   └── js/admin.js
└── data/                  # قاعدة البيانات (ملفات JSON تُنشأ تلقائيًا)
    ├── products.json · orders.json · settings.json · admin.json · secret.key
```

> **النسخ الاحتياطي:** كل البيانات موجودة في مجلد `data/`. انسخه دوريًا للحفاظ على المنتجات والطلبات.

## 📦 إدارة الطلبات

حالات الطلب: **قيد الانتظار → مؤكَّد → تم الشحن → تم التسليم** (أو **ملغى / مُرتجع**).
لكل طلب سجلّ زمني يوضّح متى تغيّرت حالته. الإيرادات تُحتسب من الطلبات **المُسلَّمة** فقط.

## 🧢 إدارة المنتجات

من **المنتجات → إضافة منتج**: الاسم، السعر، السعر القديم (للخصم)، المخزون، الوصف،
الألوان، المقاسات، شارة (مثل «الأكثر مبيعًا»)، ورفع صور (أو لصق رابط صورة).
ألغِ تفعيل أي منتج لإخفائه من المتجر دون حذفه.

## ⚙️ ملاحظات تقنية

- لا توجد أي اعتمادات (dependencies) تحتاج بناءً native — يعمل مباشرة على أي نظام.
- التخزين عبر ملفات JSON (مناسب لمتجر COD بحجم معاملات معقول). للتوسّع لاحقًا
  يمكن استبدال `src/db.js` بقاعدة بيانات (SQLite/Postgres) دون تغيير بقية الكود.
- العملة وأسعار التوصيل والتقييمات وكل النصوص قابلة للتعديل من **الإعدادات**.
